普通用户使用网页即可；开发者可以通过 HTTP API 接入其他软件。两种方式调用的是同一套核心逻辑。

普通用户

网页怎么用

  1. 打开工作台，在输入框写用途、功能、面料类型和预算。
  2. 点击“开始设计”，等待需求解析、工艺设计和成本计算。
  3. 查看推荐配方、成本和修订次数。结果只是估算，下一步仍需打样和报价。
推荐写法开发一款适用于夏季户外、成本控制在15元/米以内的速干抗菌针织面料。
系统逻辑

→

网站访问者不需要、也不应该填写 DeepSeek 密钥。密钥只由网站管理员配置在 Vercel 的加密环境变量中。

开发者

怎么调用本项目 API

向下面的接口发送 JSON。将 你的域名 换成部署后的 Vercel 地址。

curl -X POST "https://你的域名/api/v1/runs" \
  -H "Content-Type: application/json" \
  -d '{
    "user_request": "开发一款15元/米以内的速干抗菌针织面料。",
    "max_revisions": 2
  }'

成功时返回：运行状态、结构化需求、工艺方案、成本明细、修订记录和数据来源。金额使用十进制定点字符串，避免浮点误差。

接口目录

常用地址

POST /api/v1/runs提交需求并立即返回完整方案
GET /healthz检查服务是否正常
GET /api-docs自动生成的交互式调试页面
GET /api/v1/runs/{run_id}数据库模式下查询历史；Vercel 无状态演示不保存历史
重要边界

结果能不能直接生产

不能。当前成本数据标记为 is_mock: true，性能功能也是规则和模型辅助建议。必须经过打样、实验检测、工艺工程师审核和真实供应商报价。
